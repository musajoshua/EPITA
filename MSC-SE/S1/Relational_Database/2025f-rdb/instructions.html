1. Loading the Database

The database consists of three SQL scripts that must be executed in order:

  • 101_create_tables.sql - Creates the database and all tables
  • 202_insert_data.sql - Populates tables with synthetic healthcare data
  • 303_create_constraints.sql - Adds foreign keys and indexes

Loading via Command Line

-- Connect to MySQL and run each script in order
mysql -u root -p < 01_create_tables.sql
mysql -u root -p < 02_insert_data.sql
mysql -u root -p < 03_create_constraints.sql

Query 1: Patient Demographics

â˜… Simple

Objective: List all male patients born after 1990, showing their full name and city of residence, sorted alphabetically by last name.

SELECT
    CONCAT(PAT_FIRST, ' ', PAT_LAST) AS full_name,
    PAT_BRTDATE AS birth_date,
    PAT_CITY AS city
FROM patients
WHERE PAT_GENDER = 'M'
  AND PAT_BRTDATE > '1990-01-01'
ORDER BY PAT_LAST ASC
LIMIT 10;

Query 2: Organization Encounter Statistics

â˜…â˜… Intermediate

Objective: For each organization, calculate the total number of encounters and the average cost per encounter. Only include organizations with more than 50 encounters.

SELECT
    o.ORG_NAME AS organization,
    o.ORG_CITY AS city,
    encounter_count,
    avg_cost
FROM organizations o
JOIN (
    SELECT
        ENC_ORGID,
        COUNT(*) AS encounter_count,
        ROUND(AVG(ENC_TOTLCST), 2) AS avg_cost
    FROM encounters
    WHERE ENC_TOTLCST IS NOT NULL
    GROUP BY ENC_ORGID
) AS enc_stats ON o.ORG_ID = enc_stats.ENC_ORGID
WHERE encounter_count > 50
ORDER BY encounter_count DESC
LIMIT 10;

Query 3: Multi-Condition Patient Analysis

â˜…â˜…â˜… Advanced

Objective: Find patients who have been diagnosed with both a respiratory condition AND have been prescribed medication for it. Show their name, the number of respiratory conditions, total medications, and their most recent encounter date.

SELECT
    patient_name,
    respiratory_conditions,
    total_medications,
    last_encounter
FROM (
    SELECT
        p.PAT_ID,
        CONCAT(p.PAT_FIRST, ' ', p.PAT_LAST) AS patient_name,
        COUNT(DISTINCT c.CON_ID) AS respiratory_conditions,
        COUNT(DISTINCT m.MED_ID) AS total_medications,
        MAX(e.ENC_START) AS last_encounter
    FROM patients p
    JOIN conditions c ON p.PAT_ID = c.CON_PATID
    JOIN encounters e ON c.CON_ENCID = e.ENC_ID
    JOIN medications m ON e.ENC_ID = m.MED_ENCID
    WHERE c.CON_DESCRPT LIKE '%respiratory%'
       OR c.CON_DESCRPT LIKE '%bronchitis%'
       OR c.CON_DESCRPT LIKE '%asthma%'
    GROUP BY p.PAT_ID, p.PAT_FIRST, p.PAT_LAST
) AS patient_respiratory
WHERE respiratory_conditions >= 1
  AND total_medications >= 1
ORDER BY respiratory_conditions DESC, total_medications DESC
LIMIT 10;

5. Common SQL Patterns to Master

Pattern Use Case Example
INNER JOIN Combine related records from two tables Patients with their encounters
LEFT JOIN Include all records from left table, even without matches All patients, including those without encounters
GROUP BY Aggregate data into groups Count encounters per patient
Subquery in FROM Filter on aggregated values Patients with more than 5 conditions
Subquery in WHERE Filter based on another query's result Patients in cities with hospitals
CASE WHEN Conditional logic in SELECT Categorize patients by age group

6. Quick Reference: Useful Functions

String Functions

CONCAT(str1, str2)      -- Concatenate strings
UPPER(str)              -- Convert to uppercase
LOWER(str)              -- Convert to lowercase
LENGTH(str)             -- String length
SUBSTRING(str, pos, len) -- Extract substring

Date Functions

YEAR(date)              -- Extract year
MONTH(date)             -- Extract month
DATEDIFF(date1, date2)  -- Days between dates
TIMESTAMPDIFF(unit, d1, d2) -- Difference in specified unit
NOW()                   -- Current datetime

Aggregate Functions

COUNT(*)                -- Count all rows
COUNT(DISTINCT col)    -- Count unique values
SUM(col)                -- Sum of values
AVG(col)                -- Average value
MIN(col) / MAX(col)     -- Minimum / Maximum
ROUND(num, decimals)    -- Round to decimal places
